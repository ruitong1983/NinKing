## 二、标签分类（7 类）

从 v3.3 起，`ninja_data.gd` 使用 `tags: Array[String]` 替代旧的单值 `category`。每张忍者牌可拥有 **1～2 个标签**，在 Debug 弹窗和商店中以首个标签分组显示。

| 标签 | 英文 | 核心效果 |
|------|------|---------|
| **筹码** | Chips | 固定筹码加成（`add_chips`）|
| **倍率+** | +Mult | 固定倍率加法（`add_mult`）|
| **倍率X** | ×Mult | 倍率乘法（`x_mult` / 各种 ×倍）|
| **经济** | Economy | 金币产出 / 钱→数值转化 |
| **操控** | Control | 出牌次数 / 死亡保底 |
| **成长** | Scaling | 累积型效果（含重置风险）|
| **特殊** | Special | 规则变更 / 条件时窗 / 传说颠覆性机制 |

## 三、设计模式总结

### 3.1 效果设计公式

NinKing 忍者牌遵循以下标签体系（与 Balatro Joker 功能对齐）：

1. **筹码**：无条件/条件固定筹码加成（通用 n_、组别定向 n_g、特定强化 n_f）
2. **倍率+**：无条件/条件固定倍率加法（通用 n_、组别定向 n_g、成长 n_s）
3. **倍率X**：条件触发×倍率乘法（组别定向 n_g、跨组联动 n_c、喜之强化 n_x、部分经济 n_e）
4. **经济**：金币产出或钱→数值转化（经济 n_e）
5. **操控**：修改出牌次数等基础资源（忍法 n_t）
6. **成长**：满足条件→永久数值成长，高风险重置机制（成长修炼 n_s）
7. **特殊**：规则变更 n_r、传说 n_l

### 3.2 数值平衡参考

| 效果类型 | 普通级基准 | 稀有级基准 | 史诗/传说基准 |
|---------|-----------|-----------|-------------|
| 无条件倍率 | +4~+10 | — | +10~+30 |
| 无条件筹码 | +10~+20 | — | +30 |
| 条件倍率 | — | +5 | ×1.5~×2 |
| 条件筹码 | +3/张 | +40~+50 | — |
| 倍乘 | — | ×0.3 成长 | ×1.5~×4 |
| 金钱/轮 | — | +$2~$5 | 上限突破 |
| 售价 | $3~$5 | $4~$8 | $7~$14 |

### 3.3 设计原则

1. **条件越苛刻 → 数值越高**：三组严格递升(×2) > 特定牌型组合(×1.5) > 宽松条件(+5~+40)
2. **倍乘(×) > 倍率(+) > 筹码(+)**：乘法后期收益远超加法，史诗/传说级优先给 ×
3. **成长型风险与回报对等**：累积型需持续投入（修行者, 暂缓），高条件×4/×5在同花顺/豹子下爆发（天华/王座）
4. **经济/计分分轨**：前期经济卡（福神）→ 中期钱转数值（金剛力/黄金律）→ 后期高倍乘
5. **反协同设计**：强力卡常有负面效果（独尊之印前两行不计分、幻术大师仅1次出牌）
6. **规则变更牌组内互斥**（n_r 系列），强制玩家做取舍
7. **传说仅通过特殊途径获取**，商店不出售，保持稀缺性

### 3.4 与 Balatro Joker 的关键差异

| 维度 | Balatro | NinKing |
|------|---------|---------|
| 基础判定单位 | 扑克牌型（对子/顺子/同花等 9 种） | 三组排列 + 牌型（影/瞬/滅 × 9 牌型） |
| 稀有度命名 | Common / Uncommon / Rare / Legendary | 普通 / 稀有 / 史诗 / 传说 |
| 稀有度数量 | 75 / 55 / 15 / 5 | 4 / 16 / 19 / 0 |
| 经济体系 | $ 松鼠金积累 + 利息 | 无利息基础，靠忍者牌产出（利息之印突破上限） |
| 自毁机制 | 概率自毁（1/6~1/1000） | 条件重置（影非散牌/滅不满足） + 概率销毁 |
| 复制机制 | 蓝图/头脑风暴（复制相邻位） | 无直接复制，靠影武者随机增强 |
| 全局系统 | 无 | "喜"系统（9 种全局模式）联动多张忍者牌 |
