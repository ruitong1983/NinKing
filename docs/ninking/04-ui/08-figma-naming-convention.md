# NinKing Figma 命名规范

> 最后更新: 2026-06-10 | > **风格权威：**[`../05-art/16-art-direction-principles.md`](../05-art/16-art-direction-principles.md)
> Figma 设计稿统一采用少年漫画风（亮色英雄向 + 粗黑描边 + 漫画特效层）。

## 一、总体原则

| 维度 | 规则 |
|------|------|
| 语言 | 全部中文 |
| 风格 | 功能描述式 — 说清楚这是什么 |
| 组织 | 扁平命名 — 用功能名 + 类型后缀，靠 Figma 层级区分上下文 |
| 后缀 | 统一加元素类型后缀，同一个 Figma 文档内保持一致 |
| 视觉 | 所有 Frame 遵循 [`../05-art/16-art-direction-principles.md`](../05-art/16-art-direction-principles.md) §2-§4 的漫画风规范 |

## 四、特殊规则

### 有具体内容 → 用内容命名
```diff
- AC1, AC2, AC3
+ 幸运筹码能力牌, 顺子达人能力牌, 皇家礼炮能力牌
```

### 无具体内容 → 用位置/用途描述
```diff
- DecoL, DecoR
+ 能力区左侧装饰线, 能力区右侧装饰线
```

### 文本元素 → 功能 + `文本` 后缀
```diff
- GoldAmt, ShopTitle
+ 金币数量文本, 商店标题文本
```

### 卡片子元素 → 纯功能名，靠层级区分
所有能力牌/道具卡的子元素统一使用：
```
幸运筹码能力牌
├── 卡图
├── 卡图边框
├── 卡图图标
├── 名称底板
├── 卡名文本
├── 效果文本
├── 条件文本
├── 价格徽章
│   ├── 金币图标
│   └── 价格文本
└── 购买按钮
```

### 纵向排列的多文本按钮 → 上行/下行
```diff
- 出牌按钮文本1, 出牌按钮文本2
+ 出牌上行文本, 出牌下行文本
```

### 扑克牌卡面 → 花色中文 + 牌点 + 卡面
```diff
- card (A♠)
+ 黑桃A卡面
```
牌点: A, 2~10, J, Q, K
花色: 黑桃(♠), 红心(♥), 方块(♦), 梅花(♣)

## 五、与 Godot 节点命名的关系

**Figma 设计稿** 使用中文功能描述命名，用于设计沟通和团队协作。

**Godot 代码层** 使用英文前缀命名（`PNL_`/`BTN_`/`LBL_` 等，详见 `06-ui-layout-reference.md` §1），用于节点引用和 `%` unique_name。

两者不直接映射。Figma 命名关注"设计师看到什么"，Godot 命名关注"代码如何访问"。
