### 卡牌金币产出

除封印达成/利息外，以下卡牌效果可在封印中直接产金：

| 来源 | 产出 | 触发时机 |
|------|------|---------|
| 镀金增强 (Enhancement.GOLD) | +$3 / 张 | 出牌时（所在组有镀金增强的牌） |
| 金封印 (Seal.GOLD) | +$3 / 张 | 出牌时（带金封印的牌） |
| n_e01 福神 | +$2 / 喜触发 | 出牌后（每触发 1 个喜模式） |
| n_e03 俭约 | +$4 | 手替え ≤ 2 张时 |
| n_e04 利息之印 | 利息上限 +$5 | 封印达成时 |

> 所有产金效果在 `SealController._collect_play_gold()` 中统一结算。

### 钱→数值转化

| 牌 | 效果 | 封顶 |
|----|------|------|
| n_e05 金剛力 | 每 $5 持有 +1 倍率 | +10（$50 封顶） |
| n_e06 黄金律 | 每 $15 持有 ×2 | ×3（$30 封顶） |

> 这两张牌将"攒钱不花"变为一种策略选择。金剛力 $50 后不再增长（可放心消费），黄金律 $30 即达上限（×3）。

可购买:

| 次数 | 第 1 次 | 第 2 次 | 第 3 次 | 第 4 次 | 第 5+ 次 |

## 存档与解锁

### 双存档模型

| 存档 | 路径 | 生命周期 |
|------|------|---------|
| **Run Save** | `user://ninking_run_save.json` | 封印开始时 checkpoint 写入；死亡/通关时自动删除 |
| **Progress Save** | `user://ninking_progress.json` | 永久保留（解锁记录 + 统计） |

### 永久死亡规则

```
封印开始时 → checkpoint save
    │
    ├─ 死亡 (GAME_OVER) → record_run_result(敗北)
    │                       delete_run()
    │                       → 回主菜单，只能"新游戏"
    │
    ├─ 通关 (VICTORY)   → record_run_result(胜利)
    │                       delete_run()
    │
    └─ 中途退出           → checkpoint 保留
                            → 下次启动可"继续"（从当前封印开头续玩）
```

### 持久化数据 (Progress Save)

| 字段 | 类型 | 说明 |
|------|------|------|
| `unlocked_decks` | Array[String] | 已解锁牌组（初始: standard, night, sun） |
| `deck_best_barriers` | Dictionary | `{deck_name: best_barrier_reached}` |
| `total_runs` | int | 总局数 |
| `total_wins` | int | 总胜场 |
