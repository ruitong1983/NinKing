# NinKing 图像素材生成方案 — 豆包 AI 生成

## 概述

> **目标：** 用豆包（Doubao）AI 图像生成工具，分阶段产出 NinKing 项目的全部图像素材，替换现有的麻将占位图。
> **风格：** 复古像素风（16-bit 游戏机风格）+ 扑克牌主题 + CRT 滤镜兼容
> **总素材量：** 约 25 张核心生成图 + Godot 内拼装编号牌

## 生成策略

### 为什么不用豆包逐张生成 52 张牌？

豆包生成 52 张独立卡牌会导致：风格不一致、花色图案不统一、工作量大。采用**"模板 + 拼装"**策略：

```
豆包生成核心素材（~20-25 张）
    │
    ├── 卡牌底板（1 张）— 所有牌共用
    ├── 花色符号（4 张）— ♠ ♥ ♦ ♣
    ├── 人头牌角色（12 张）— J/Q/K × 4 花色
    └── Ace 装饰牌（4 张）— 每花色独特设计

Godot 程序化拼装
    │
    ├── 编号牌 2-10：底板 + 数字(像素字体) + 花色符号 × N
    └── 卡牌背面：底板背面设计（1 张）
```

**最终产出：** 52 张正面 + 1 张背面 = 完整扑克牌组

### 豆包提示词通用规则

每条提示词遵循以下结构（中文 + 英文关键词混合，增强精准度）：

```
[风格锚定] [主体描述] [色彩方案] [构图要求] [技术参数]
```

## 豆包生成实操指南

### 推荐参数设置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 模型 | 豆包 Seedream 4.0 / 即梦 | 优先选最新版 |
| 比例 | 按素材表格指定 | 卡牌类 2:3，UI 类按需 |
| 风格 | 像素风 / Pixel Art | 关键词中英双语强调 |
| 清晰度 | 最高 | — |
| 负面词 | `模糊, 抗锯齿, 渐变, 3D渲染, 写实, 照片` | 确保像素纯度 |

### 批次生成建议

```
批次 1（花色符号 4 张）— 快速验证风格，10 分钟
批次 2（卡牌底板 2 张）— 确定卡牌 UI 基调，10 分钟
批次 3（人头牌 JQK 12 张）— 核心美术资产，分 4 组按花色跑，1 小时
批次 4（Ace 4 张）— 装饰牌，20 分钟
批次 5（UI 组件 10 张）— 界面素材，分 2 组，40 分钟
批次 6（装饰元素 5 张）— 润色阶段，可选
```

### 后处理流程

每批生成后执行：

1. **像素纯度检查：** 放大 400% 确认边缘无模糊/抗锯齿
2. **调色板统一：** 使用 Aseprite 或 GraphicsGale 统一色板（建议 32 色以内）
3. **尺寸裁剪：** 严格按规格裁剪，去除多余透明边距
4. **格式转换：** 保存为 PNG-8（索引色）减小体积
5. **导入 Godot：** 放入 `res://assets/images/` 对应子目录

## 附录 A：编号牌 2-10 的 Godot 拼装方案

编号牌（36 张 = 9 个数字 × 4 个花色）不用豆包逐张生成，而是在 Godot 内用程序拼装。

### 拼装元素

| 元素 | 来源 | 说明 |
|------|------|------|
| 卡牌底板 | 豆包生成 `card_frame_front.png` | 白色牌面+边框 |
| 花色符号 | 豆包生成 `suit_*.png`（4 张） | 48×48 像素 |
| 像素数字 2-10 | 像素字体 Godot Label | 用 Zpix / Press Start 2P 字体渲染 |

### 拼装布局（160×240 画布）

```
┌──────────────────────┐
│ 2♠              ♠2  │  ← 左上角数字+花色, 右下角翻转
│                      │
│                      │
│        ♠ ♠           │  ← 2 个花色符号（N 个按牌面值排列）
│                      │
│                      │
│                      │
└──────────────────────┘
```

### 花色符号排列规则

| 牌面数字 | 符号排列（自上而下） |
|----------|---------------------|
| 2 | 上下各 1 个 |
| 3 | 上中下各 1 个 |
| 4 | 2×2 网格 |
| 5 | 2-1-2 三行 |
| 6 | 3×2 网格 |
| 7 | 2-1-2-2 四行 |
| 8 | 3-2-3 三行（中央偏移）|
| 9 | 3-3-3 三行 |
| 10 | 4-3-3 三行（左上角改为 "10"）|

### 实现方式

推荐用 `card_renderer.gd` 工具脚本，在编辑器内批量生成 36 张编号牌的 PNG，存入 `assets/images/cards/numbers/`。或在游戏运行时实时渲染到 `TextureRect`。

## 附录 C：像素字体推荐
