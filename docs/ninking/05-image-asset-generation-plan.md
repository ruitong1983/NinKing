## 1. 概述

### 1.1 目标

用豆包 AI 分批生成 NinKing 项目的核心图像素材，配合 Godot 程序化拼装系统，覆盖全部视觉缺口，替换现有 FanKing 麻将主题占位图。

### 1.2 游戏视觉定位

| 维度 | 方向 |
|------|------|
| **主风格** | 少年漫/热血漫（Shonen Manga） |
| **子风格** | 忍者 × 扑克牌 Roguelike |
| **色调** | 亮色英雄向 — 高饱和 accent + 开阔明快的底色 |
| **核心视觉语言** | 粗黑描边 · 速度线 · 集中线 · 拟声词 · 半调网点 · cel 质感 |
| **核心视觉锚点** | 漫画「忍」字（Logo）、粗描边花色符号（牌面）、手里剑 + 网点（卡背）、8 属性亮色 |

### 1.3 设计决策（2026-06-10 烧烤确认 + 漫画风更新）

| # | 议题 | 决策 | 理由 |
|---|------|------|------|
| 1 | 卡牌正面风格 | **仅花色符号 4 张** AI 生成 | 牌面程序绘制保持不变（§7.1 原则文档），仅替换花色纹理为漫画粗描边版 |
| 2 | 背景图风格 | **亮色英雄向 + 漫画网点** | 与 8 属性亮色面板统一；亮底让卡牌/UI 更突出 |
| 3 | 45 张忍者牌 | **模板化拼装** | 4 底板 × ~10 图标 → Godot 拼出 45 张（+2 deferred），成本最低且风格统一 |
| 4 | 10 Boss 视觉 | **卷轴底板 + 10 符号** | 通用底板 + 各 Boss 标志性符号，入场仪式感拉满 |
| 5 | 游戏图标 | **黑桃 A + 忍刀交叉（漫画粗描边风）** | 最强牌面 + 忍者身份，一眼识别 |
| 6 | 消耗品/道具 | **3 底板 + 少量图标** | Phase B 功能未落地，先模板化保持商店不空 |
| 7 | Logo 装饰 | **漫画粗描边「忍」字** | 游戏视觉锚点，比英文标题独特 |
| 8 | 阶段排期 | **三阶段（P0→P1→P2）** | 先做玩家感受最明显的第一印象链路 |
| 9 | 卡牌背面风格 | **漫画网点 + 粗黑描边 +「忍」字** | 像素手里剑十字星 → 漫画半调网点 + 和风纹样（§7.2 原则文档） |
| 10 | CRT 滤镜 | **已移除** | 与亮色漫画风冲突（§4.4 原则文档） |

## 2. 阶段计划总览

### 2.1 三阶段排期

```
Phase P0 ────→ Phase P1 ────→ Phase P2
第一印象         游戏核心         锦上添花
(~10 张 AI)     (~30 张 AI)     (~8 张 AI)
```

| 阶段 | 覆盖范围 | 玩家感受 | AI 张数 | 拼装张数 |
|------|----------|----------|---------|----------|
| **P0** 第一印象 | 启动→主菜单→第一眼牌面 | ⭐⭐⭐⭐⭐ | ~10 | 0 |
| **P1** 游戏核心 | 牌桌→商店→Boss | ⭐⭐⭐⭐ | ~25 | 45 忍者 + 10 Boss |
| **P2** 锦上添花 | 商店道具区 | ⭐⭐ | ~8 | 25 消耗品 |

### 2.2 为什么 P0 先做

玩家从"双击桌面图标 → 主菜单 → 发牌看到手牌"这条路径，是**每局必走、第一印象决定留存**的黄金 30 秒：

- `icon.png`：桌面/任务栏第一眼
- `launch_bg.png`：启动画面，情绪铺垫
- `logo_deco.png`：过关转场遮罩，仪式感
- 4 花色符号：牌面核心元素，每局看上千次

P0 只需 **10 张图**，就能让"启动→主菜单→看牌"整条链路焕然一新。

## 3. 豆包 AI 生成实操指南

### 3.1 推荐参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 模型 | 豆包 Seedream 4.0 / 即梦 | 优先选最新版 |
| 比例 | 按素材表格指定 | 卡牌类 5:7，背景 16:9，图标 1:1 |
| 风格关键词 | `少年漫画风` `粗描边` `cel-shading` `高饱和` `半调网点` | 中英双语强调 |
| 清晰度 | 最高 | 漫画线条模糊 = 废图 |
| 负面词 | `写实, 照片, 3D渲染, 光滑渐变, 像素风, 8-bit, 暗黑, 恐怖` | 确保漫画纯度 |

### 3.2 提示词通用模板

```
[风格锚定] + [主体描述] + [色彩方案] + [构图要求] + [技术参数]

示例：
少年漫画风，日式漫画，粗描边，cel-shading，[主体]，[颜色]，[构图]，
半调网点纹理，高饱和配色，纯平面2D，[视角]。
尺寸 [W]×[H] 像素。背景[透明/颜色]。
```

### 3.3 后处理流程

每批生成后执行：

1. **漫画风格检查：** 确认粗描边清晰、cel-shading 块面分明、网点纹理可见
2. **描边统一：** 使用 Aseprite 或 GraphicsGale 统一描边粗细（主体 2-3px 仿手绘）
3. **调色板统一：** 色板控制在 32 色以内，高饱和 accent + 明亮底色
4. **尺寸裁剪：** 严格按规格裁剪，去除多余透明边距
5. **格式转换：** 保存为 PNG（支持半透明网点叠加）
6. **导入 Godot：** 放入 `res://assets/images/` 对应子目录
7. **.import 检查：** 确认 import 设置 Filter=false, Mipmaps=false（保持漫画线条锐利）

### 3.4 批次生成建议

```
批次 1（花色符号 4 张）— 快速验证风格，10 分钟
批次 2（背景 2 张 + icon + logo）— 确定视觉基调，30 分钟
批次 3（忍者底板 + 图标）— 核心美术资产，1 小时
批次 4（Boss 卷轴 + 符号）— Boss 视觉，40 分钟
批次 5（消耗品底板 + 图标）— 润色，20 分钟
```

**关键词：** `少年漫画风` `扑克牌` `黑桃` `粗描边` `符号`

少年漫画风，日式漫画，扑克牌黑桃（Spade）花色符号，纯黑色填充（#1A1A1A），

## 7. Godot 模板拼装系统设计

> **目标：** 用最少的 AI 生成素材，通过 Godot 程序化拼装，产出全部功能卡牌的视觉呈现。
> **核心理念：** 底板 + 图标 + 数据驱动 = 最终卡牌

### 7.1 忍者牌拼装系统

#### 拼装公式

```
NinjaCard 视觉 = ninja_frame_{rarity}.png   ← AI 生成 4 张（稀有度边框+网点底板）
               + icon_{effect}.png           ← AI 生成 10 张（效果图标）
               + 名称 Label                  ← Godot 字体渲染
               + 描述 Label                  ← Godot 字体渲染
               + 价格标签                    ← Godot 字体 + 金币图标
               + 槽位占用指示器              ← 程序绘制（V6）
```

#### 数据驱动映射

每张忍者牌的数据（来自 `ninja_data.gd`）包含：

```gdscript
{
    "id": "n_s01",
    "name": "喜鹊",
    "rarity": "common",          # → ninja_frame_common.png
    "icon": "star",              # → icon_star.png
    "effect_text": "喜触发时额外+1筹码",
    "cost": 4,
    "category": "xi"             # 喜相关
}
```

拼装时：
1. 根据 `rarity` 选择底板纹理
2. 根据 `icon` 选择图标纹理
3. `name` / `effect_text` / `cost` 用 Label 渲染
4. 组装成完整的 `NinKingCard` → `TextureRect`

#### 实现方式

在 `ninking_card.gd` 或新建 `ninja_card_renderer.gd` 中实现：

```gdscript
# 伪代码
func render_ninja_card(ninja_data: Dictionary) -> void:

    var icon = load("res://assets/images/ninjas/icons/icon_%s.png" % ninja_data.icon)
    # 设置 TextureRect 层级
    # frame_texture.texture = frame
    # icon_texture.texture = icon
    # name_label.text = ninja_data.name
    # desc_label.text = ninja_data.effect_text
    # cost_label.text = "$%d" % ninja_data.cost
```
