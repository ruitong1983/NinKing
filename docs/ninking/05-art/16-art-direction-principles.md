## §2 色彩体系 — 八属性

### 2.1 属性→Ante 映射

> 每个 Ante 绑定一个忍术属性。属性决定该 Ante 内所有 Blinds 的背景色、accent 色、粒子色调。

| Ante | 属性 | 称谓 | 色彩印象 |
|------|------|------|----------|
| 1 | **火** | 壱·火 | 暖焰橙红 — 温暖、入门 |
| 2 | **水** | 弐·水 | 清流淡蓝 — 柔和、适应 |
| 3 | **風** | 参·風 | 微风翠绿 — 清新、自由 |
| 4 | **雷** | 肆·雷 | 暖光金黄 — 明亮、积极 |

## §3 UI 组件规范

### 3.1 面板（Panel）

| 属性 | 规范 |
|------|------|
| **圆角** | 8-12px（治愈系要大圆角，不要硬角） |
| **描边** | 无硬描边，用柔和阴影替代。Kenney 纹理面板不额外描边 |
| **底色** | 暖米 `#F5F0E8`（`panel_beige`/`panel_beigeLight`）为主，属性 `panel` 色为辅 |
| **阴影** | 面板下方 4px 偏移 + `#00000015` 柔和投影（模拟纸片浮在纸面上的层次感） |

> **例外 — 分割装饰线：** 少数需要视觉分隔的场景（商店舞台顶部边界、信息块分隔），使用 **1-2px 暖灰半透明线**，不适用粗墨线。

### 3.2 按钮（Button）

| 状态 | 表现 |
|------|------|
| **normal** | Kenney 暖纸纹理（`buttonRound_beige`/`buttonLong_brown`），圆润造型，深褐字 `#3D2B1A` 或白字。无硬描边。 |
| **hover** | 底色调亮/高亮 + 轻微 scale 1.03（`GlobalTweens.scale_pop`）。温暖放大感而非冲击感。 |
| **pressed** | 底色调暗 10% + content 下移 1px（纸按钮的"被按下"感）。 |
| **disabled** | 暖灰色底 + 浅灰字 `#AAAAAA`。 |

### 3.3 三墩区分

> 替代粗描边递进方案。采用**温和区分**，用细微色差+图标的"轻/中/重"区分，而非粗线冲击。

| 墩 | 表现 | 标签 |
|----|------|------|
| **影** | 细微暖灰底纹，1px 半透明边框 | 「影」|
| **瞬** | 浅色底 + 微弱属性色 glow | 「瞬」|
| **滅** | 底色调深 + 柔和光晕 border | 「滅」|

**约束满足时：** 三道标签同时点亮为属性色 + 柔和光粒从标签散开。
**约束违规时：** 违规段标签变灰 + 小「×」叠印。

### 3.4 通用原则

- **无硬描边：** 全局不使用 `#1A1A1A` 粗描边轮廓。用 Kenney 暖纸纹理自带的视觉效果。
- **圆角优先：** 所有交互元素优先采用 8px+ 圆角。
- **纸质感：** 大面积底色区域优先使用 Kenney `panel_beige`/`panel_beigeLight` 九宫格纹理，营造暖纸面质感。
- **柔和阴影：** 所有可操作元素（按钮、卡片、弹窗）保持柔和投影，强化纸片层次感。

## §4 治愈系柔和特效层

> 特效层是叠加在 UI 之上的独立视觉层，仅在特定情绪节点触发。治愈系特效强调**温柔感动**而非冲击力。

### 4.1 柔和光晕绽放（Soft Glow Burst）

| 属性 | 规范 |
|------|------|
| **触发时机** | Boss 揭晓 / 封印达成 / 双喜+ 触发 / 稀有牌出现 |
| **视觉效果** | 从中心向外扩散的柔和暖色光晕（径向渐变），持续 0.5-0.8s |
| **实现** | `ColorRect` 全屏叠加 + 暖色光晕纹理 + `GlobalTweens.fade_in_out()` |
| **配合音效** | `SB.BOSS_REVEAL` / `SB.SEAL_CLEAR` |

### 4.2 轻飘浮粒子（Gentle Float）

| 属性 | 规范 |
|------|------|
| **触发时机** | 发牌结束 / 卡牌交换 / 换牌 / 过关转场 |
| **视觉效果** | 暖色光点/花瓣状粒子轻柔飘浮，速度慢、轨迹柔和 |
| **实现** | `CPUParticles2D` + `TweenFX.gentle_float()` |
| **配合音效** | `SB.SWAP` / `SB.DEAL` 柔和版 |

### 4.3 温和文字弹出（文字ポン）[P2]

| 属性 | 规范 |
|------|------|
| **触发时机** | 討伐按钮按下（「ぽん」）/ 喜触发（「わぁい」）/ Boss 揭晓（「おっ」）/ 封印失败（「ざんねん…」） |
| **视觉效果** | 柔和圆体文字从触发点弹出，scale 0.8→1.1→1.0 + 淡出 1.2s。不加粗，不冲击。 |
| **实现** | `TweenFX.gentle_text_pop(node, text, position)` — P2 |
| **字体要求** | 与 UI 主字体 LXGW WenKai Medium 保持一致，不额外使用粗手写体 |

### 4.4 已移除项

> **CRT 滤镜 — 已移除**（旧少年漫画方向已移除）。治愈系不适用任何扫描线/像素类滤镜。

## §5 音效设计原则

### 5.1 风格方向

| 维度 | 旧（Chiptune+和风） | 新（治愈动画 SFX） |
|------|---------------------|-------------------|
| **参考来源** | FC 游戏、8-bit 音源 | 治愈系动画、日常系作品音效 |
| **操作音** | 方波/三角波短促音 | 轻柔 tap / click / ふわっ（轻柔感） |
| **情绪节点** | 太鼓/铃铛/筝 | 温和 sting / gentle whoosh / sparkle / ゆるい |
| **BGM** | Chiptune + 和乐器 | 治愈 BGM 短循环 + 属性变奏，温暖宁静 |

### 5.2 分层策略

```
日常操作层（高频，< 0.3s）
  ├─ 发牌/交换/选牌/按钮  →  轻柔短促音（纸感/轻 tap）
  └─ 计分跳动               →  轻 tick（溫和版）

情绪节点层（低频，0.3-2s）
  ├─ Boss 揭晓               →  温和 suspense + 轻鼓点
  ├─ 喜触发                  →  温柔 sparkle chime + 浮起感
  ├─ 封印达成/失败           →  温暖 victory chime / 轻柔叹息
  ├─ 商店进入/离开           →  温和 door chime / 轻 whoosh
  └─ 忍者牌激活              →  柔和 energy pulse

环境层（循环，低音量）
  ├─ BGM（属性变奏）         →  治愈系 OST 短循环
  └─ 商店 BGM                →  温暖 relaxing jingle
```

### 5.3 关键词速查（给素材搜寻用）

```
日语: 癒し系効果音, 優しい, ふんわり, 軽やか, かわいい
英语: healing sfx, gentle click, soft tap, cozy game sound,
      warm chime, gentle whoosh, sparkle soft, relax jingle
```

## §8 VFX / 粒子

### 8.1 治愈系粒子预设

> 在现有 `ParticlePool` 基础上新增治愈系预设，**废弃旧少年漫画的集中线/速度线预设**。

| 预设 | 实现 | 优先级 |
|------|------|--------|
| `heal_glow` — 柔和光晕粒子 | 新纹理（64×64 暖色径向渐变 PNG）+ `ParticlePool` 预设 | **P1** |
| `heal_petal` — 花瓣飘落 | 花瓣形 PNG + `ParticlePool` 预设，轻柔弧线下落 | **P1** |
| `heal_float` — 暖光点上浮 | 圆形光点 PNG + `CPUParticles2D`，缓慢上浮 | **P1** |
| `heal_sparkle` — 温和闪光 | 小星形光点 + 短暂闪烁（替代集中线爆发） | **P2** |

### 8.2 温和文字弹出 [P2]

```
TweenFX.gentle_text_pop(node: Node, text: String, position: Vector2)
  - Label 实例化 → LXGW WenKai Medium → 从 position 弹出
  - Tween: scale 0.8→1.1→1.0 (0.2s ease_out) + modulate.a 1→0 (1.2s fade)
  - 完成后 queue_free()
```

### 8.3 移除项
