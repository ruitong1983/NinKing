## §1 核心风格定义

### 1.1 风格定位

| 维度 | 方向 |
|------|------|
| **主风格** | 少年漫/热血漫（Shonen Manga） |
| **子风格** | 忍者 × 扑克牌 Roguelike |
| **色调** | 亮色英雄向 — 高饱和 accent + 开阔明快的底色 |
| **核心视觉语言** | 粗黑描边 · 速度线 · 集中线 · 拟声词 · 半调网点 · cel 质感 |

### 1.2 风格关键词

```
中文: 少年漫画 热血 粗描边 速度线 集中线 拟声词 网点 cel-shading 半调
英文: shonen manga, action lines, speed lines, halftone dots, cel shading,
      thick outlines, manga screentone, comic fx, bold
```

### 1.3 参考作品

| 作品 | 参考维度 |
|------|---------|
| 『我的英雄学院』游戏 UI | 亮色英雄向配色、速度线叠层 |
| 『街头霸王 6』漫画滤镜 | 集中线爆发、墨迹飞溅、拟声词弹出 |

## §3 UI 组件规范

### 3.1 面板（Panel）

| 属性 | 规范 |
|------|------|
| **圆角** | 6-8px（漫画不是像素，不要硬角） |
| **描边** | 2-3px 仿手绘描边（粗细略有不均的 G-pen 感），深色 `#1A1A1A` |
| **底色** | 当前属性 `panel` 色 + 可选半调网点叠加（5-8% 不透明度） |
| **阴影** | 面板下方 4px 偏移 + `#00000020` 柔和投影（模拟漫画格的层次感） |

> **例外 — 漫画格分割线：** 商店底部舞台的顶部边界、关卡过渡的全幅分割线等需要模仿漫画分格线的场景，使用 **4px 直角粗墨线 `#1A1A1A`**，不适用圆角规范。目的是制造"翻到下一格"的漫画阅读体验，不是 UI 窗口感。

### 3.2 按钮（Button）

| 状态 | 表现 |
|------|------|
| **normal** | 属性 accent 色底 + 深黑描边 + 圆角 8px。白色粗体文字。三级尺寸：L(4px描边), M(3px描边), S(2px描边)。 |
| **hover** | 底色调亮 10% + 描边加粗至 3px。鼠标悬浮时附加轻微 scale 1.03（`GlobalTweens.scale_pop`）。 |
| **pressed** | 底色调暗 15% + content 下移 2px（漫画按钮的"被按下去"感）。 |
| **disabled** | 灰色底 `#CCCCCC` + 浅灰描边 `#999999`。 |

**按钮拟声词点缀（P2）：** 重要按钮（討伐/换牌/Boss揭晓）pressed 时可选弹出小型拟声词 Label（`TweenFX.comic_text_pop()`）。

### 3.3 三墩区分

> 替代像素风的 1px→2px→3px 边框递进。

| 墩 | 边框表现 | 标签 |
|----|---------|------|
| **影** | 1px 虚线描边（漫画中"轻"的笔触） | 「影」 |
| **瞬** | 2px 实线描边 + 微弱属性色 glow | 「瞬」 |
| **滅** | 3px 粗描边 + 速度线底纹（3 条短线平行装饰） | 「滅」 |

**约束满足时：** 三道标签同时点亮为属性 accent 色 + 集中线从标签向卡牌汇聚。  
**约束违规时：** 违规段标签变灰 + 小「×」叠印（漫画错误标记）。

### 3.4 通用原则

- **描边颜色：** 全局统一 `#1A1A1A`（漫画墨色），不使用彩色描边（accent 通过底色表达）
- **线宽变化：** 允许在同一面板上存在 1px / 2px / 3px 多种描边宽度（模拟手绘笔压），但同一层级保持统一
- **网点叠加：** 大面积底色区域可选叠加 5-10% 半调网点纹理（`screentone_pattern.png`），增强"印刷品"质感

## §4 漫画特效层

> 特效层是叠加在 UI 之上的独立视觉层，仅在特定情绪节点触发，不常驻。

### 4.1 集中线（Focus Lines）

| 属性 | 规范 |
|------|------|
| **触发时机** | Boss 揭晓 / 封印达成 / 双喜+ 触发 / 稀有牌出现 |
| **视觉效果** | 从屏幕边缘向中心汇聚的放射状线条，持续 0.3-0.5s，线条密度 12-20 条 |
| **实现** | `ColorRect` 全屏叠加 + 集中线纹理 + `GlobalTweens.fade_in_out()` |
| **配合音效** | `SB.BOSS_REVEAL` / `SB.SEAL_CLEAR` |

### 4.2 速度线（Speed Lines）

| 属性 | 规范 |
|------|------|
| **触发时机** | 发牌结束 / 卡牌交换 / 换牌瞬身 / 过关转场 |
| **视觉效果** | 3-5 条水平或斜向粗短线，跟随运动方向 |
| **实现** | `CPUParticles2D` 挂载 → `TweenFX.speed_line_trail()` (P0) |
| **配合音效** | `SB.SWAP` / `SB.DEAL` / `SB.REDRAW_POP` |

### 4.3 拟声词弹出（Comic SFX Pop）[P2]

| 属性 | 规范 |
|------|------|
| **触发时机** | 討伐按钮按下（「討つ！」）/ 喜触发（「喜！」）/ Boss 揭晓（「来た！」）/ 封印失败（「無念…」） |
| **视觉效果** | 粗手写体文字从触发点弹出，scale 0.5→1.2→1.0 + 淡出 1.5s |
| **实现** | `TweenFX.comic_text_pop(node, text, position)` — P2 独立功能 |
| **字体要求** | 手写体中文字体（见 §10 附录 B — 暂缓） |

### 4.4 CRT 滤镜 — 已移除

> **决策：移除。** 亮色英雄漫画风追求干净线条 + 高饱和色彩，CRT 扫描线直接冲突。
>
> 移除清单见 §9 实施清单。

## §5 音效设计原则

### 5.1 风格方向

| 维度 | 旧（Chiptune+和风） | 新（动画 SFX） |
|------|---------------------|-----------------|
| **参考来源** | FC 游戏、8-bit 音源 | 日本动画音效库、动画电影 |
| **操作音** | 方波/三角波短促音 | 清脆 tap / click / swish（动画常用） |
| **情绪节点** | 太鼓/铃铛/筝 | 戏剧性 sting / whoosh / impact / sparkle |
| **BGM** | Chiptune + 和乐器 | 动画风 OST 短循环 + 属性变奏 |

### 5.2 分层策略

```
日常操作层（高频，< 0.3s）
  ├─ 发牌/交换/选牌/按钮  →  短促清脆音（保留 Chiptune 精确反馈感）
  └─ 计分跳动               →  light tick

情绪节点层（低频，0.3-3s）
  ├─ Boss 揭晓               →  动画 suspense sting + drum roll
  ├─ 喜触发                  →  sparkle chime + swell
  ├─ 封印达成/失败           →  动画 victory sting / sad string
  ├─ 商店进入/离开           →  door chime / transition whoosh
  └─ 忍者牌激活              →  energy pulse

环境层（循环，低音量）
  ├─ BGM（属性变奏）         →  anime OST style 短循环
  └─ 商店 BGM                →  轻松明亮 jingle
```

### 5.3 关键词速查（给素材搜寻用）

```
日语: アニメ効果音, 漫画効果音, バトル, 衝撃, 閃光, 緊迫, 勝利, 敗北
英语: anime sfx, manga sound effect, battle whoosh, impact hit,
      dramatic sting, victory jingle, suspense drum, sparkle chime,
      cartoon pop, comic sfx
```

## §7 卡牌设计

### 7.1 扑克牌牌面 — 保持不变

> **决策：保持经典扑克牌面。** 牌面是功能性界面元素，过度风格化影响手牌判断速度。
>
> - 程序绘制：奶油白底 + 8px 圆角 + 18px 角标 + 56px 中央花色 + 右下 180° 镜像
> - 花色颜色：红桃♥/红方♦ `#CC1A1A`，黑桃♠/梅花♣ `#1A1A1A`
> - 花色符号：由 P0 AI 生成的 4 张漫画风粗描边符号替换 Unicode 文字（仅替换符号纹理，不改变牌面布局）

### 7.2 卡牌背面 — 重绘

| 属性 | 旧（像素风） | 新（漫画风） |
|------|-------------|-------------|
| **底纹** | 像素交叉纹 | 漫画半调网点 + 和风纹样 |
| **描边** | 2px 像素金边 | 3px 粗黑描边（G-pen 感） |
| **中心图案** | 像素手里剑十字星 | 漫画「忍」字 + 手里剑粗描边插图 |
| **色调** | 深蓝底 + 金边框 | 深色底（保持与亮色 UI 的对比） + 赤红/金色点缀 |
| **实现** | `CardBackGenerator` 程序绘制 | 豆包 AI 生成纹理 → `CardBackGenerator` 替换纹理引用 |

### 7.3 忍者牌拼装（模板体系）

> 拼装体系不变（4 底板 × ~10 图标 → Godot 拼出 45 张），只替换素材风格。
>
> - 底板：按 §6.2「忍者牌底板」漫画风重绘
> - 图标：按 §6.2 通用风格指南重绘
> - 详情见 `05-image-asset-generation-plan.md` §7（Godot 拼装系统设计）
