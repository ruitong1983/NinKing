# NinKing 核心玩法设计 — 忍者牌 × 比鸡

> **游戏定位：** 忍者主题扑克 Roguelike。将比鸡（9 张牌 → 3 组 × 3 张，升序约束）与 Balatro 式闯关体系融合。
> **设计原则：** 保留比鸡的排列 Puzzle DNA + Balatro 的系统深度 + 忍者世界观包装
>
> **📂 卡牌/关卡/经济细节已拆分至独立文档:** `11` 忍者牌 · `12` 消耗品 · `13` 关卡与Boss · `14` 经济与进程

# Part 0: 核心定义

## 一句话

> **手牌 9 张，排列成 3 组 × 3 张（影≤瞬≤滅升序约束），AI 辅助排列，三组同时计分。搭载忍者牌、附魔卡、星图卡、喜、秘仪五大系统，闯过 8 Antes × 3 Blinds 通关。**

## 核心参数速查

| 参数 | 值 |
|------|-----|
| 手牌 | 9 张 |
| 每组 | 3 张（影/瞬/滅，共 3 组） |
| 排列约束 | 牌力(影) ≤ 牌力(瞬) ≤ 牌力(滅) — **硬约束** |
| 出牌次数 | **3** / Blind |
| 换牌次数 | **2** / Blind，每次换 **1~3 张** |
| 交换位置 | 不限次（玩家通过交换覆盖 AI 排列） |
| 忍者牌槽位 | 5（可通过 Negative 版本扩展） |
| 初始金币 | **$8** (v3.1: $4→$8) |
| 利息 | 每 $5 余额 +$1，上限 $5 |
| 牌组 | 5 种（标准/暗夜/赤阳/混沌/龙脉） — Phase D |
| 存档 | 纯永久死亡，仅保留解锁进度 |

# Part 1: 核心循环

## 1.1 Blind 内循环

```
发 9 张手牌
    │
    ├── AI 自动排列 → 展示 3 组 + 预期忍気 + 喜触发提示
    │       │
    │       ├── [交换] 拖动牌在组间/组内移动（不限次）
    │       │       覆盖 AI 排列，实时约束验证
    │       │
    │       ├── [换牌] → 选 1~3 张丢弃 → 重抽 → AI 重排
    │       │       每 Blind 2 次
    │       │
    │       └── [出牌] → 计分 → 9 张进手替札 → 抽 9 张 → 下一轮
    │
    └── 忍気达标 → 过关（进入商店）
        出牌次数归零且未达标 → 失败（永久死亡）
```

### v3.1 交互决策

| 操作 | 消耗 | 解决的问题 |
|------|------|-----------|
| 交换 | 不限次 | 微调排列结构，覆盖 AI 建议 |
| 换牌 | 2 次/Blind, 1~3 张/次 | 改善手牌质量 |
| 出牌 | 3 次/Blind | 确认计分 → 下一轮 |

> **设计决策:** 取消手动交换后玩家只剩「确认出牌」→ 决策太薄。两者都保留：AI 给建议，玩家有交换权覆盖，同时有换牌改善手牌质量。

## 1.2 关卡结构

```
8 Antes，每个 Ante = 3 Blinds (Small / Big / Boss)
```

# Part 2: AI 自动排列

## 2.1 算法

- 暴力枚举 1680 种排列（C(9,3)×C(6,3)）
- 过滤合法排列（满足影≤瞬≤滅约束）
- 在合法排列中选分数最高的
- 性能: < 1ms（GDScript 单帧）

## 2.2 牌力编码

```
strength = hand_type × 10000 + high_rank × 100 + mid_rank + suit_tiebreak

hand_type: 豹子=6, 同花顺=5, 同花=4, 顺子=3, 对子=2, 散牌=1
suit_tiebreak: ♠=0.4, ♥=0.3, ♦=0.2, ♣=0.1
```

## 2.3 规则型忍者牌的 AI 适配

持有规则变更型忍者牌时，AI 对应调整过滤条件：

| 忍者牌 | 效果 | AI 行为 |
|--------|------|---------|
| n_r02 均衡之印 | 三组必须同牌型 | 过滤条件改为 h_type == m_type == t_type |
| n_r03 独尊之印 | 滅 ×2，影/瞬 ≥ 对子 | 过滤条件改为影/瞬牌型 ≥ 对子 |

> n_r01 自由之印已删除——解除约束移除核心 Puzzle，无趣。

## 2.4 AI 排列的玩家覆盖

- AI 给出最优排列建议
- 玩家可通过**交换**（不限次）手动调整
- 约束校验器实时反馈：绿箭头 = 满足，红箭头 = 违规
- 出牌按钮仅约束满足时可用

# Part 3: 计分系统

## 3.1 公式（扁平模型）

```
Group_i_chips = group_i_card_chips + group_i_base_hand_chips
Group_i_mult  = group_i_base_hand_mult

TotalChips = Σ(Group_i_chips for i=1,2,3) + Σ(ninja_add_chips) + joy_chips + card_ench_chips
TotalMult  = Σ(Group_i_mult for i=1,2,3)  + Σ(ninja_add_mult) + card_ench_mult
Final      = TotalChips × TotalMult × ∏(ninja_x_mult) × ∏(joy_x_mult) × ∏(card_x_mult)
```

所有 ×mult 效果统一压入栈尾，乘法满足交换律 → 无顺序依赖。

## 3.2 牌面筹码

| 牌面 | 筹码 |
|------|------|
| A | 11 |
| K, Q, J | 10 |
| 10~2 | 面值 |

## 3.3 3 张牌型

|------|---------|---------|
| 豹子 | 100 | 8 |
| 同花顺 | 50 | 5 |
| 同花 | 30 | 4 |
| 顺子 | 20 | 3 |
| 对子 | 10 | 2 |
| 散牌 | 5 | 1 |

# Part 4: 喜系统

## 4.1 机制

- 每次出牌后自动检测 9 张牌的全局模式
- 每个喜独立检测，触发即叠加（扁平模型）
- 无分类互斥、无上限、无优先级纠结

## 4.2 9 种喜

| # | 喜名 | 条件 | 效果 |
|---|------|------|------|
| 1 | 全黑 | 9 张全 ♠/♣ | ×1.5 |
| 2 | 全红 | 9 张全 ♥/♦ | ×1.5 |
| 3 | 全顺（一条龙） | 9 张点数连续 | ×2.0 |
| 4 | 全同花 | 9 张同一花色 | ×2.5 |
| 5 | 四张 | 手中有 4 张同点数 | +50 chips |
| 6 | 三清 | 三组全是同花（花色可不同） | ×1.3 |
| 7 | 三顺清 | 三组全是同花顺 | ×3.0 |
| 8 | 顺清打头 | 影为同花顺 | ×1.5 |
| 9 | 全三条 | 9 张可组成 3 副三条 | ×4.0 |

> 注：「全顺」和「全三条」概率极低（52 张标准牌组中类似中彩票），不应围绕它们设计忍者牌。
